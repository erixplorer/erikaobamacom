## 1. My immediate thought on Affinity became completely free

A few months ago, <a href="https://www.affinity.studio/" target="_blank">Affinity</a> became free. My immediate thought on this move was "It can't take over the creative industry, especially in Japan, unless it adds the vertical text direction feature".

<img src="/src/images/article/7_affinity-v3/1_vertical-writing-use-cases-in-japan.webp" alt="Vertical writing use cases in Japan" />

Don't get me wrong. I personally liked Affinity V2 for the excellent usability. On the other hand, Canva's UX felt rather confusing, and it lacked some essential features. But it had a vertical text feature for a long time.  

So… Why not? Now the new Affinity is integrated with Canva. It could be possible to use the vertical writing…

## 2. The Facts

There are a couple of reasons why I don't think Affinity will take over the creative industry.  

### Lack of Vertical Writing Compatibility

This is *not* the biggest reason, but it's still critical.

According to <a href="https://realsound.jp/tech/2023/09/post-1446012.html" target="_blank">a 2023 interview with Affinity CEO</a>, the vertical text feature seemed almost promised to be introduced in the near future. Meanwhile, Japanese users invented a few workarounds: 
- <a href="https://s-suzume.booth.pm/" target="_blank">Vertical-writing exclusive fonts</a> and <a href="https://gist.github.com/fdiary/5dbade3da21bed6d7a8e0abf8ff697b8" target="_blank">a FontForge script</a> to build
- <a href="https://cs5.xyz/blog/2019/09/affinityv4.html" target="_blank">A method to manually adjust</a> paragraph or character styles

But, all these workarounds could only be temporary solutions. There are <a href="https://www.w3.org/TR/jlreq/?lang=en" target="_blank">so many typography rules</a> to follow to develop a proper vertical typing feature. It requires tedious work, and it isn't easy even for a million-dollar company.

### Lack of RTL Compatibility

Not only vertical writing, but Affinity also doesn't support <a href="https://forum.affinity.serif.com/index.php?/search/&q=vertical%20writing" target="_blank">right-to-left writing</a>. This is another typography feature that has been requested for years, but no progress has been made so far. In fact, the UI is also not compatible with RTL languages.

<img src="/src/images/article/7_affinity-v3/8_affinity-rtl-compatibility.webp" alt="RTL compatibility issues possted on the Affinity official forum" />
<figcaption>RTL compatibility is also a thing</figcaption>


### Performance Issue

Typing features aside, I found a big downside of unifying three applications into one in its RAM usage. In short, Affinity consumed much more memory than Illustrator for simple tasks. 

<img src="/src/images/article/7_affinity-v3/9_affinity-v3-performance-issue.webp" alt="Affinity V3 performance issue" />



### Target Users by Region

Affinity has been a common choice for freelancers and small companies in the Western world. So far, this trend hasn't changed so much. Professionals still recommend this suite mostly to <a href="https://www.reddit.com/r/graphic_design/comments/1pwu0z9/should_i_switch_to_affinity_in_2026/" target="_blank">freelancers who don't collaborate with other companies</a>.

I thought: Perhaps the situation is now different in Russia, since Adobe has been sanctioning the country for years…? Not at all. <a href="https://www.youtube.com/watch?v=Pmvea2e8WnI&t=13114" target="_blank">People still recommend Adobe</a> for the same reason.  

(By the way, I do understand some basic Russian. But I don't understand how people manage to use Adobe there... I'm glad to discover how things are working there, if you know about it...)

In Japan? Well, I see it's <a href="https://cad-kenkyujo.com/affinity/" target="_blank">treated more like a content creator tool</a>. If you are just stepping up from Canva or if you rarely use the text tool, it's recommended. 


## 3. What's Important to Think: The Nature of Design Projects

### Collaboration

While performance issues can be temporary, and typography issues can remain regional, the nature of design work is not really transformable, even with the AI. Because, more often than not, design projects thrive with collaboration. A modern collaboration requires technical compatibility.  

<img src="/src/images/article/7_affinity-v3/10_collaborative-design-workflow.webp" alt="Collaborative design workflow" />


A long time before Adobe Creative Cloud, we had many more <a href="https://en.wikipedia.org/wiki/Comparison_of_desktop_publishing_software" target="_blank">"industrial standard" design tools</a>. Basically, the more standard tools, the more compatibility issues. Even if you chose to be Team Adobe, you couldn't be completely free from the version compatibility issue. 

<img src="/src/images/article/7_affinity-v3/11_adobe-version-compatibility.webp" alt="Adobe version compatibility" />
<figcaption>Source: <a href="https://www.graphic.jp/direct_upload" target="_blank">Graphic Netprint</a></figcaption>

Of course, this issue was much smaller than the software incompatibility. Adobe Suite also covered <a href="https://webdesignerdepot.com/adobe-creative-suite-5-brings-sexy-back/" target="_blank">a wide range of creative fields</a>, so it actually was the most convenient suite available.  

In 2013, Adobe had finally solved this incompatibility problem with Creative Cloud, then eventually dominated the market… But, not entirely.


### The Cases Adobe Competitors Won

For example, Adobe isn't really dominating the Web development field.

- <a href="https://www.figma.com/" target="_blank">Figma</a> had beaten <a href="https://helpx.adobe.com/support/xd.html" target="_blank">XD</a> a long time ago. Now you have 100% of chance to encounter Figma as a designer for the web medium
- There was also <a href="https://brackets.io/" target="_blank">Brackets</a> around the same time as <a href="https://code.visualstudio.com/" target="_blank">Visual Studio Code</a> became the most popular code editor. Now VS Code keeps winning over.
- <a href="https://www.adobe.com/products/dreamweaver.html" target="_blank">Dreamweaver</a> is now integrated with <a href="https://github.com/" target="_blank">GitHub</a>, but I think the workflow has replaced by modern website platforms and website generators…?

I see the Web development field has a lot of tools coming and going in a very short time, but what I'm trying to say here is that different creative fields have different niches and industrial standards. The same goes for the 3D industry, too.

<img src="/src/images/article/7_affinity-v3/12_standard-tools-by-industry.webp" alt="Standard tools by industry" />


## Final Thoughts

Nowadays, everything comes as a groundbreaking innovation or a threat to your current workflow, job, or whatever stability you want to protect.  

Now, let's pause for a second. Didn't humanity come across this kind of change every few decades? How realistic is a new tool to take over the whole industry across the world instantly, only because it has a shiny new feature? Yes, it happens, but only when it has enough reasons and evidence to convince people to switch over.

<img src="/src/images/article/7_affinity-v3/13_product-lifecycle.webp" alt="Product lifecycle" />
<figcaption>Source: <a href="https://uxdesign.cc/four-rules-for-how-design-influences-technology-44b2c1948135" target="_blank">Four rules for how design influences technology</a></figcaption>


So… Don't panic. Instead, try to have a flexible mindset and skills to thrive in any industrial change. The same applies to your personal life. (The AI is coming anyway, so…)

This way, you can stay calm and focus on learning the things that will actually be useful or important in the near future.
